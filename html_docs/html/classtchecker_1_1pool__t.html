Detailed Description

template<class T>
class tchecker::pool_t< T >

Pool allocator with collection.

Template Parameters
T: type of allocated objects. Should derive from tchecker::make_shared_t<Y> for some Y
Note
Pools allocate blocks of memory. Each block contains a fix number of chunks. A chunk stores an object of type T. All chunks have the same fixed size alloc_size. A block contains a fixed alloc_nb chunks. The size of a block is alloc_nb * alloc_size + sizeof(void *). The extra size for a pointer is used to maintain a simple linked list of blocks.
The pool is NOT thread-safe

Constructor.

Parameters
alloc_nb: number of chunks in a block (allocation unit)
alloc_size: fixed size of chunks
Precondition
alloc_nb >= 1
Postcondition
initialized to empty pool that allocates memory by blocks of alloc_nb chunks, each chunk of size max(alloc_size, MIN_ALLOC_SIZE) bytes
Note
alloc_size should be determined by a call to tchecker::allocation_size_t<T>::alloc_size(). This class should in turn be specialized for type T
the pool allocates at least MIN_ALLOC_SIZE bytes per chunk. Hence, the actual allocation size is max(alloc_size, MIN_ALLOC_SIZE). The MIN_ALLOC_SIZE bytes are needed to maintain a list of free chunk, while keeping the value of the reference counter of each chunk untouched.
extra sizeof(void *) bytes are allocated for each block to maintain a list of allocated blocks
Exceptions
std::invalidargument when the precondition is not satisfied
